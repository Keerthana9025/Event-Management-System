# Event-Management-System
**Objective:**
Create a backend system using Java, Spring Boot, and PostgreSQL to manage events, sessions, and participants.
**Key Requirements:**
   1) **Implement APIs to perform CRUD operations for:**
        - Events (title, description, date, location, category)
        - Sessions within each event (topic, speaker, start time, end time)
        - Session materials limited to:
            - PDF documents (e.g., speaker slides, agendas)
            - Video links (e.g., recorded sessions, promo videos)
    2)  Allow filtering events by category or date.
    3)  Support participant registration for events via APIs.
    4) **Include endpoints to:**
        - Add, update, delete, and list events, sessions, and materials
        - Register participants and list their registered events
    5)  Thoroughly test with unit and integration tests.
    6)  (Optional) Add an endpoint to generate a PDF schedule for a specific event.
**Guidelines:**
     1) Create a new repository and raise PR on daily tasks.
     2) Design DB schema and API structure.
     3) Define the API structure and endpoint naming conventions.
     4) Ensure clean code, proper exception handling, and clear documentation.
